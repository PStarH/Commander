方法论

测试架构：TELOSOrchestrator + TopologyRouter 完整管道，AgentRuntime 边界替换为模拟真实 LLM 延迟（5-20ms）和 token 消耗（50-200 tokens）的 fake runtime。

韧性模拟：fake runtime 模拟 2% 瞬态失败率 + 1 次重试，有效失败率降至 0.04%，模拟生产环境的 circuit breaker + retry 行为。

任务类型：8 种任务覆盖 FACTUAL/ANALYSIS/RESEARCH/CODING/REVIEW/DEBUG/CREATIVE/DATA 场景。

并发级别：1/5/10/20/50 并发，每级别 10 轮迭代，共 860 个任务。

判定标准：failure rate < 1% 为 PASS。Latency P99 < 500ms 为 SLO。
