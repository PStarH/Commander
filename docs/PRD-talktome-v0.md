# TalkToMe PRD v0

## 产品定位

**一句话**: TalkToMe 是一个自托管的、以个人聊天记录为核心的"树洞分身"。

当你（或想你的人）累了、想说话时，可以打开一个链接，和一个尽可能像你、但诚实直接的分身聊天。

**关键信念**:
- 它不是效率工具，不帮你排 todo
- 它不是心理医生，也不假装解决一切
- 它就是一个基于你真实语言和经历训练出的"你味道"的树洞，**真实反馈 > 安慰话术**
- 数据优先保存在你自己的机器上，安全性和隐私是第一原则

---

## 目标用户与场景

### 目标用户
- 对「数字分身」感兴趣、想用真实聊天记录训练一个自己的树洞分身的人
- 创作者 / 独立开发者 / 重度 IM 用户：聊天和贴文已经非常多，希望这些东西不只是留在历史里
- 想要给自己或亲近的人留一个"以后还能跟我说句话"的入口的用户

### 典型场景

1. **自用树洞**
   - 半夜睡不着、不想刷社交、不想麻烦别人
   - 打开 TalkToMe，向"另一个自己"倒垃圾、复盘一天、骂世界
   - 期望得到的是：被理解 + 适度诚实的反馈，而不是套话鼓励

2. **关系里的"缓冲区"**
   - 和某人吵完架、不知道怎么表达的时候
   - 先跟 TalkToMe 说一遍，看"另一个自己"会怎么说，帮你整理情绪和表达方式

3. **未来维度**
   - 用户希望"万一哪天我不在了"
   - 亲友还能通过这个链接，看到一些他生前留下的东西，并且以那种熟悉的语气再聊几句

## 核心价值主张 (v0)

### 1. 高保真自我
通过导入你自己的聊天记录/帖子，构建一个比"通用大模型人格"更接近你的分身。

重点不是技能，而是：
- 语言习惯
- 态度
- 价值观偏好

### 2. 本地优先、安全可控
- TalkToMe v0 设计为**本地/自托管部署**，数据默认存在本机
- 即便需要调用云模型，也只在你允许的前提下发送**必要片段的摘要**，而非全量原始聊天记录

### 3. 真实反馈，而非虚假安慰
- 不强行用"你很棒""你可以的"粉饰一切
- 更像你会对自己/朋友说的话：
  - 该说你在逃避的时候，会点出来
  - 但不会用攻击性的方式伤你

## v0 功能范围

### 4.1 数据导入
支持导入以下格式（v0 从本地文件开始）：
- 导出的聊天记录（JSON / HTML / txt）
- 个人长文/帖子（Markdown / HTML / txt）

简单的**源配置**：
- 每个数据源可以命名（例如"和 A 的聊天""Twitter 帖子""个人日记"）
- 用户可以选择：
  - 要全部导入
  - 只导入某段时间
  - 某些源标记为"永不导入"

导入后在本地做：
- 去重、简单清洗（去掉系统提示、无意义 log）
- 基本的元信息标注：时间、对话对象、来源平台

### 4.2 本地存储与索引（Rust 核心）
使用 Rust 实现一个常驻服务（daemon）负责：
- 加密存储（例如加密 sqlite 或加密 JSON 文件）
- 建立简单的倒排索引 / embedding 索引（用于按主题/语气检索片段）
- 提供本地 HTTP API：
  - `POST /ingest`：导入新文本
  - `POST /chat`：给定当前对话消息，返回一个"分身回复"
  - `POST /search`：按关键词/时间检索历史片段

### 4.3 对话体验（文本）
一个简洁的 Web 界面（或先用简单终端/CLI UI）：
- 左侧：对话历史
- 右侧：输入框
- 上方明显标注：
  > "你正在和 TalkToMe（基于你自己的数据训练的分身）聊天。"

对话行为（后端逻辑）：
- 每次用户发消息时：
  1. 在本地索引中检索与你当前话题/情绪相关的历史片段
  2. 选取少量片段拼成"记忆上下文"
  3. 传给 LLM（本地或云）生成回复，带上 system prompt：
     - 你的说话风格
     - 你对"真实反馈"的偏好

### 4.4 "真实反馈"基调控制
在配置里让用户选择/调节一个简单参数（v0 文案级别）：
- "希望你对我：温和一点 / 直接一点 / 非常诚实"

system prompt 示例约束：
- 优先帮助用户看清现实和自己模式
- 批评行为，不攻击人格
- 避免对严重心理危机给出具体行动建议，鼓励用户寻求人类帮助

## 非目标（v0 明确不做）

- 不做任何生产力功能（任务管理、日程提醒等）
- 不接入社交平台 API 做"偷偷抓数据"，所有数据必须是用户手动导出/上传
- 不尝试伪装成"用户本人仍然活着"骗亲友
- 不提供公开的多人聊天室形态（v0 是**一人一树洞**）

---

## 技术架构（高层）

### Rust 后端
主要模块：
- `ingest`：处理导入数据
- `store`：加密存储、简单查询
- `index`：文本 embedding / 关键词索引
- `api`：本地 HTTP 服务

### 对话逻辑
一个对话代理（可以先用 Node/Python 写，之后也可 Rust）：
- 从 TalkToMe 后端取"相关记忆片段"
- 调用 LLM 生成回复（带"人格 prompt" + few-shot 示例）
- 返回给前端

### 前端
v0 可用最简单的 Web 界面（Vite + React/Tailwind），只跑在本机。

---

## 安全与隐私设计

- **默认离线 / 本地部署**，所有数据存在用户机器加密文件中
- 如需使用云 LLM：
  - 明确展示"即将发送的内容（摘要）"，让用户确认或关闭云模式
  - 最好默认对长历史做摘要后再发，而不是原文大片上传
- 提供一键"彻底删除数据"的选项

## 未来方向（超出 v0）

- **声音分身**：给用户上传的语音/视频做 voice clone
- **给指定亲友入口**："如果你想他，可以来这里跟 TalkToMe 说两句"
- **更丰富的"回忆触发器"**：按日期、地点、特定 tag 触发某些记忆
- **和 Commander 整合**：
  - Commander 管打仗
  - TalkToMe 管"你脑子里那支军队的情绪和反思"
