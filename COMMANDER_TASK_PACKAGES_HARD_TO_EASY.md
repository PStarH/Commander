# Commander 任务包 — 从难到易排序

> 用于按能力分配：高级工程师拿最难，新人拿最容易

---

## 🏗️ Very Hard (4-8 周)

### 1. Pkg3 — AgentRuntime 上帝对象分解
- **重要性**: P1 🔴
- **给谁**: 首席/高级工程师，熟悉整个架构的人
- **核心**: 4,571 行 `agentRuntime.ts` 拆成 5-8 个模块，76 个单例替换为 DI
- **工时**: 4-8 周
- **风险**: ⚠️ 必须增量进行，不能阻塞其他包

---

## 🔬 Hard (1-3 周)

### 2. Pkg7 — Quality Gates 重写
- **重要性**: P2 🟡
- **给谁**: ML/AI 工程师
- **核心**: 正则表达式质量门替换为 LLM-as-judge 或语义嵌入评估
- **工时**: 1-2 周

### 3. Pkg12 — 沙箱与插件系统
- **重要性**: P1-P2 🔴🟡
- **给谁**: 安全工程师 + 后端工程师
- **核心**: 实现真实代码执行沙箱、完成插件系统集成、TEE 并行执行
- **工时**: 2-3 周

---

## 🧪 Medium-Hard (1-2 周)

### 4. Pkg5 — 集成测试与 SLO 测量
- **重要性**: P1 🔴
- **给谁**: 测试/QA 工程师 + DevOps
- **核心**: 端到端集成测试、SLO CI 门控、负载测试、失败注入
- **工时**: 1-2 周

---

## 📊 Medium (3-7 天)

### 5. Pkg6 — 真实监控与可观测性
- **重要性**: P1 🔴
- **给谁**: DevOps/后端工程师
- **核心**: 5 个虚假 health check 替换为真实实现、结构化错误日志、告警规则
- **工时**: 3-5 天

### 6. Pkg8 — Provider 系统加固
- **重要性**: P2 🟡
- **给谁**: 后端工程师
- **核心**: 能力文档化、Perplexity/Replicate 工具调用修复、超时隔离、健康检测
- **工时**: 3-5 天

### 7. Pkg9 — 自我优化修复
- **重要性**: P2 🟡
- **给谁**: ML/AI 工程师
- **核心**: 修复 `analyzeModelPerformance()` 返回空 Map、移除浪费的 LLM 调用
- **工时**: 3-5 天

### 8. Pkg10 — 延迟与性能优化
- **重要性**: P2 🟡
- **给谁**: 高级工程师
- **核心**: SubAgentExecutor 背压、同步 I/O 移到 worker threads、自适应原子化器
- **工时**: 3-7 天

### 9. Pkg2 — 多租户声明修正
- **重要性**: P0 ⛔
- **给谁**: 任何 TypeScript 开发者
- **核心**: 标记 API 为 alpha、更新文档、添加运行时警告
- **工时**: 2-5 天

### 10. Pkg13 — SLO 测量基础设施
- **重要性**: P1-P2 🔴🟡
- **给谁**: DevOps
- **核心**: CI 中添加 4 个 SLO 测量（恢复、故障转移、补偿、DLQ）
- **工时**: 3-5 天

### 11. Pkg14 — 存储与持久化加固
- **重要性**: P2-P3 🟡🟢
- **给谁**: 后端工程师
- **核心**: WAL 轮转、DLQ 轮转、连接池、统一内存系统
- **工时**: 2-4 天

### 12. Pkg15 — 基础设施与 CI/CD 加固
- **重要性**: P0-P3 ⛔→🟢
- **给谁**: DevOps
- **核心**: Windows 构建修复、CD 验证、告警规则、Docker Compose 生产检查
- **工时**: 2-4 天

---

## ⚡ Easy-Medium (1-3 天)

### 13. Pkg1 — 紧急安全与健康检查修复
- **重要性**: P0 ⛔
- **给谁**: 任何人
- **核心**: 修复正则注入、修复健康检查虚假数据、添加速率限制、SQLite 告警
- **工时**: 2-3 天

---

## 🧹 Easy (1-2 天)

### 14. Pkg4 — 死代码清理
- **重要性**: P1 🔴
- **给谁**: **新人/实习生最佳入口**
- **核心**: 删除 actor/、inspectorAgent、frameworkIntegration、adaptiveOrchestrator、重复 company.ts、遗留拓扑别名
- **工时**: 1-2 天
- **安全**: 只需确保 `tsc --noEmit` 通过

### 15. Pkg11 — 配置与文档清理
- **重要性**: P2 🟡
- **给谁**: **新人/实习生最佳入口**
- **核心**: 同步 schema topology 枚举、CLI init 自动创建 .env、更新 README 功能状态
- **工时**: 1-2 天
