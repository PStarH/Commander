chore: format normalizeExpected.ts (Prettier)

Remote CI hard-failed the entire CI matrix on a Prettier format check
against packages/core/src/observability/normalizeExpected.ts. The
file shipped from commit 765b41430 (dual-classifier refactor that
extracted normalizeForMatch primitives) had a closing `}` followed
immediately on the same line by `export interface GradedClassification {`
— Prettier wants a blank line between them.

Because the format gate runs early in the CI matrix (across all OS x
Node combos), every downstream gate was blocked: vitest,
GAIA benchmark, SupplyChainScanner, and pre-commit hook PATH-fix
validation could not be exercised by remote CI. Local validation
in commit 765b41430's pre-push review was green but did not include
a Prettier format check — closing that gap is a followup.

Fix: `npx prettier --write packages/core/src/observability/normalizeExpected.ts`.
Pure formatting, no semantic impact. After this commit, the Prettier
check passes for this file and the matrix can run end-to-end.

Note: a broader `npx prettier --check` reveals ~60 other latent format
violations across packages/core/src, packages/core/tests/observability,
and scripts/. Those are unrelated to this gate-blocking issue and will
be addressed in a separate chore:format-sweep commit — keeping the
fixes scoped per the user's explicit ask.
