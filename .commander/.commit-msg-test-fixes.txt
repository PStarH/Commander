## Cascade of failures in traceContextBridge.test.ts

The single biggest finding was that traceContextBridge.test.ts was
effectively BROKEN as a unit — vite:oxc parses fail-fast on the first
syntax error, so each fix revealed the next. Four latent defects in
one file surfaced in turn:

1. L88: `it('writes trace context into a shallow copy without mutating
   the caller's dict', ...)` — unescaped ASCII apostrophe inside
   single-quoted test name. Fix: switch to double quotes.

2. L102: `it('does not overwrite an existing same-key entry silently —
   keeps the caller's value', ...)` — same defect, different test, only
   surfaced after L88 because of fail-fast parsing. Fix: double quotes.

3. L160: `it('RESERVED_TRACE_CONTEXT_KEY is exactly '__traceContext'',
   ...)` — a clever-but-broken attempt to embed `__traceContext` inside
   the test name using string-literal concatenation. The parser sees
   `'RESERVED_TRACE_CONTEXT_KEY is exactly '` as one string then
   `__traceContext` as an identifier. Fix: switch to double-quoted
   outer string, single quotes inside consume verbatim.

4. L161: `expect(RESERVED_TRACE_CONTEXT_KEY).toBe('__traceContext')`
   — stale literal assertion. The actual export is
   `RESERVED_TRACE_CONTEXT_KEY = 'cmdr_trace_context'` per
   `traceContextBridge.ts` L48 (with rationale JSDoc at L28 stating
   the `cmdr_` prefix is deliberately chosen). The test was asserting
   an OLD/ALTERNATE name from `agentInbox.ts`, a separate inbox-level
   system that uses a different convention. Fix: assert
   `'cmdr_trace_context'` to match source-of-truth.

Combined effect: traceContextBridge.test.ts now compiles AND its
named-constant assertion holds. The four fixes are documented
together because the cascade is impossible to bisect post-hoc.

## Forward-compatible companion to commit 70477154

commit 70477154 added the empty-expected regression guard — cases
without ground truth now correctly route to `graded: false, score: 0`
to prevent the silent UNGRADED-to-INCORRECT collapse. Two callers
needed forward-compat fixes to interact cleanly with the new guard:

5. `packages/core/src/observability/autoScorer.ts` — `traceToTarget()`
   now sets `expected: 'live-trace-quality-evaluation'` as a sentinel.
   Live traces have no dataset-style ground truth; the sentinel lets
   them bypass the empty-expected regression guard and route through
   the LLM judge for trace-quality assessment. The judge prompt sees
   the sentinel header and distinguishes quality scoring from
   accuracy scoring. This preserves the guard for dataset cases
   while letting live-trace scoring continue to function. A comment
   in the source documents the design and the rationale.

6. `packages/core/tests/observability/evalHttpEndpoints.test.ts`
   L287 — added `expected: 'ok'` (matching the case executor's
   `output: 'ok'`) to the two dataset cases. Without `expected`,
   the regression guard correctly routes both cases to
   `graded: false, score: 0` and `passed` stays at 0. With the
   matched `expected`, both cases legitimately count as PASSED
   and the test's intent (POST /datasets/:id/run returns 202;
   polling /experiments/:id returns the completed run with
   passed=2) is restored.
