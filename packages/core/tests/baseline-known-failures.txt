### cpuWorkerPool.ts — logger variadic strictness (pre-existing, latent)

```
src/runtime/cpuWorkerPool.ts(109,90): error TS2345: Argument of type 'string'
    is not assignable to parameter of type 'Error'.
src/runtime/cpuWorkerPool.ts(148,102): error TS2345: Argument of type 'unknown'
    is not assignable to parameter of type 'Error | undefined'.
```

Root cause: the `getGlobalLogger().error(category, message, ...)` interface has a
strictly-typed variadic last-arg signature of `Error | undefined`. The worker
error/exit callbacks pass a `string` (from `err.message ?? String(err)`) and a
`catch`-bound `unknown`. Both are pre-existing; the P1.B cleanup rewrote only
the `try { this.workers[index]?.terminate(); } catch { ... }` block, which
compiles clean (`reportSilentFailure(_silentE_, ...)` is happy with `unknown`).

To fix in a future PR: tighten the catch/handler bindings to `Error` (e.g.
`(err: Error) => { ... }` with an explicit `instanceof` guard outside) OR widen
the logger-error variadic signature to `unknown`.

`ultimate/orchestrator.ts`. Pre-dates the Phase 1 (storage) refactor, the
